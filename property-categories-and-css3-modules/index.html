Property Categories and CSS3 Modules

How to make things look pretty.

Last week we talked about how to get things in the right place with CSS properties that relate to layout. Today we will talk about how to skin the elements and get the look and feel we want.

Before we dive in, let's talk about pseudo-elements.

Pseudo-elements

Pseudo-elements let you inject elements into your markup from your stylesheet.

<p>Hello, world!</p>

See the Pen CSS Power-ups - Examples - Pseudo-elements by Matt Swensen (@mjswensen) on CodePen.

These are the types of pseudo-elements:

  • ::before
  • ::after
  • ::first-letter
  • ::first-line
  • ::selection
  • ::backdrop
::before and ::after are somewhat misleading. The CSS Power-ups homepage has an example of ::selection. ::backdrop might be used for modal overlays in the future.

CSS Evolvement Process

Step 1: CSS specifications are developed by the World Wide Web Consortium (the W3C).

The W3C is made up of a full-time staff as well as member organizations, and is lead by Tim Berners-Lee.

Step 2: Browser creators (Microsoft, Google, Mozilla) can choose to implement support for the specifications in their browsers.

Step 3: Web developers can use those features in their web pages.

This process is why we have web pages today that look different than the ones that existed in the 90's.

Sometimes browser vendors implement support for a feature before it becomes a W3C Recommendation.


ul {
  -moz-columns: 2; /* for Firefox */
  -webkit-columns: 2; /* for Chrome, Safari */
  columns: 2; /* future-proofing */
}

To determine when vendor prefixing is necessary, I use caniuse.com.

When they do this, they prepend the feature with their proprietary prefix (e.g., "-webkit-"). Put the spec-compliant version last. Given that CSS3 is always under development, tools like caniuse or prefixer are invaluable.

CSS3 Modules

Current progress on module development.

The W3C has split CSS3 into multiple "modules," to make the progression of CSS more modular and agile (as opposed to simply developing the whole version update at once). This way, modules can advance toward recommendation status at different paces, or can be abandoned altogether without as much overhead.
